# Music Player Application

## 📌 Overview
A Spotify-like music streaming application where users can browse songs, create playlists, listen to music with a custom audio player, view lyrics, and discover new music through recommendations. Features include audio visualization, queue management, shuffle/repeat modes, and social sharing.

## ✨ Features
- User authentication and personalized profiles
- Browse music by genre, artist, album, or mood
- Search songs, artists, and albums
- Custom audio player with controls (play, pause, skip, volume)
- Progress bar with seek functionality
- Queue management (add, remove, reorder songs)
- Shuffle and repeat modes (off, one, all)
- Create, edit, and delete playlists
- Like/favorite songs and artists
- Recently played history
- Audio visualization (waveform, spectrum analyzer)
- Lyrics display with sync (optional)
- Download songs for offline listening (optional)
- Share songs and playlists
- Artist pages with discography
- Album view with tracklist
- Dark mode with theme customization
- Keyboard shortcuts for playback
- Responsive design for desktop and mobile

## 🛠️ Tech Stack
- **Frontend**: React/Next.js with TypeScript or Electron (desktop app)
- **Audio Player**: Howler.js, React Player, or HTML5 Audio API
- **State Management**: Zustand or Redux Toolkit
- **Backend**: Node.js + Express or Next.js API Routes
- **Database**: PostgreSQL with Prisma or MongoDB
- **Authentication**: NextAuth.js or Firebase Auth
- **File Storage**: AWS S3, Cloudinary, or Bunny CDN for audio files
- **Audio Visualization**: Web Audio API or Wavesurfer.js
- **Styling**: Tailwind CSS with Framer Motion
- **Lyrics**: Musixmatch API or Genius API
- **Deployment**: Vercel (web) or Electron (desktop)

## 📊 Difficulty Level
**Intermediate** - Requires understanding of audio playback, Web Audio API, complex state management for playback queue, file streaming, and real-time UI updates. Perfect for learning media applications and audio processing.
